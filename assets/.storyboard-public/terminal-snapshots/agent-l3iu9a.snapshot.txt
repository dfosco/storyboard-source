  type identifier don't match.

  The real problem: there is no schema for widget types anywhere agents can find.

   - .agents/skills/canvas/SKILL.md shows examples (sticky-note, markdown, agent) but doesn't enumerate the full
  set or document each type's props shape.
   - .agents/data/ only has primer-octicons.json.
   - The server accepts any string as type without validation, so unknown types create "ghost" widgets that persist
   in JSONL but render nothing — silent failure.

  What would fix this (analogous to the octicons list):

   1. A generated .agents/data/canvas-widgets.json listing every widget type + required/optional props, derived
  from the renderer registry in packages/storyboard/src/internals/canvas/widgets/.
   2. A scripts/gen-widget-types.mjs to regenerate it.
   3. A note in AGENTS.md / canvas SKILL: "Before creating a widget, verify the type exists in
  .agents/data/canvas-widgets.json."
   4. Bonus: server-side validation that rejects unknown types with a clear error instead of silently creating
  ghost widgets.

  Want me to ship that?
