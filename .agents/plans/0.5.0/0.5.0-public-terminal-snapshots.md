# 0.5.0 — Public Terminal Snapshots

## Problem

Public terminal snapshots (`assets/.storyboard-public/terminal-snapshots/<widgetId>.snapshot.json`) are not being generated for any terminals. The private buffers (`.storyboard/terminal-buffers/`) work correctly, but the `assets/` public counterpart directory remains empty. This means:

1. **Production builds have no terminal snapshots** — `TerminalReadWidget` can't show anything in production
2. **No git-committed record** of terminal output exists

Additionally, the current snapshot format is JSON-only. The `dump-session.sh` utility in unified-vision produces a much more human-readable text format. We should adopt that style for a `.snapshot.txt` companion file alongside the JSON.

## Root Cause Analysis

### Suspected primary cause: `hasTmux` gate

`startSnapshotCapture(snapshotOpts)` is only called when `hasTmux === true` (line 1142). If tmux is unavailable, **no snapshots are ever captured** — neither private buffers nor public.

However, the private `.buffer.json` and `.buffer.txt` files ARE being written in unified-vision, which means `hasTmux` is true and `captureSnapshot()` IS running. The bug is therefore in the public snapshot write path specifically.

### Suspected secondary cause: silent error swallowing

The public snapshot write block (lines 397-404) wraps everything in a bare `catch {}`:

```js
try {
  mkdirSync(sDir, { recursive: true })
  writeFileSync(snapshotTmpPath, JSON.stringify(snapshotData, null, 2), 'utf8')
  renameSync(snapshotTmpPath, snapshotPath)
} catch {
  try { if (existsSync(snapshotTmpPath)) unlinkSync(snapshotTmpPath) } catch {}
}
```

If this throws (permissions, path resolution, cross-device rename, etc.), the failure is completely invisible.

### Diagnosis steps needed

Before implementing the fix, we need to confirm the root cause by:

1. Adding `devLog` calls to the public snapshot write path to surface errors
2. Checking if the `assets/.storyboard-public/terminal-snapshots/` directory exists and is writable in a running client project
3. Verifying `isWidgetPrivate()` returns `false` for typical terminals (it should — `widgetProps.private` is never set by `handleConnection`)

## Plan

### Todo 1: Add diagnostic logging to `captureSnapshot`

**File:** `packages/core/src/canvas/terminal-server.js`

Add `devLog` calls at key decision points in `captureSnapshot()`:
- Log when `isWidgetPrivate` returns true (skipping public snapshot)
- Log when the public snapshot write succeeds (with path)
- Log the actual error when the public snapshot write fails (replace bare `catch {}` with `catch (err) { devLog().logEvent('error', '...', { error: err.message }) }`)
- Log when the private buffer write succeeds/fails similarly

This makes the snapshot pipeline observable during development.

### Todo 2: Fix non-tmux snapshot capture

**File:** `packages/core/src/canvas/terminal-server.js`

Remove the `if (hasTmux)` gate around `startSnapshotCapture` (line 1142). The rolling buffer is populated by `ptyProcess.onData()` regardless of tmux vs direct pty, and `captureSnapshot` already handles the case where `tmux capture-pane` fails (line 311 — catches and falls through to rolling buffer only).

Similarly, remove the `hasTmux` guard from `stopSnapshotCapture` calls in the `close` and `error` handlers (lines 1173, 1188).

This ensures snapshot capture works for both tmux and non-tmux sessions.

### Todo 3: Investigate and fix the public snapshot write failure

**File:** `packages/core/src/canvas/terminal-server.js`

After adding logging (Todo 1), run the dev server in a client project and observe what happens. Likely causes to check:

- **Path resolution**: `publicSnapshotDir()` uses `process.cwd()` — verify this resolves correctly from the Vite server-plugin context
- **Directory creation**: `mkdirSync(sDir, { recursive: true })` — verify `assets/.storyboard-public/terminal-snapshots/` can be created
- **Cross-device rename**: `renameSync(tmpPath, finalPath)` fails on cross-device moves. Both paths should be in the same directory, but verify the `.tmp` file and final file are on the same filesystem

Once the root cause is identified, fix it and verify public snapshots appear in `assets/.storyboard-public/terminal-snapshots/`.

### Todo 4: Add `.snapshot.txt` — readable text format for public snapshots

**File:** `packages/core/src/canvas/terminal-server.js`

After writing the JSON snapshot, also write a human-readable `.snapshot.txt` companion file to the same public directory. Format inspired by the `dump-session.sh` output:

```
SESSION: terminal-abc123 | my-terminal
CANVAS:  design-system
BRANCH:  main
TIME:    2026-04-29T10:00:00.000Z

================================================================================
SCREEN
================================================================================

<ANSI-stripped pane content>

================================================================================
SCROLLBACK (last 60s)
================================================================================

<ANSI-stripped scrollback content>
```

Key design decisions:
- **ANSI-stripped** — the text file is for humans and git diffs, not terminal rendering
- **Header block** with identity info — easy to scan which terminal this belongs to
- **Section separators** using `===` lines — matches the dump-session.sh style
- **Same write pattern** as the JSON: tmp file + atomic rename
- **Same private gate** — skip writing `.snapshot.txt` when widget is private
- **Same tilde rename** — `~<widgetId>.snapshot.txt` for private widgets

### Todo 5: Update the Vite build plugin to include `.snapshot.txt`

**File:** `packages/react/src/vite/data-plugin.js`

The `storyboard-terminal-snapshots` Vite plugin (line 1407) currently only emits `.json` files. Update it to also copy `.snapshot.txt` files into the build output at `_storyboard/terminal-snapshots/`.

The `TerminalReadWidget` doesn't need changes — it only reads JSON for rendering. The `.txt` files are for:
- Human browsing in the repo
- Git diff visibility
- Agent consumption

### Todo 6: Add `.snapshot.txt` to the server-plugin unwatch list

**File:** `packages/core/src/vite/server-plugin.js`

The watcher already unwatches `assets/.storyboard-public/terminal-snapshots` (line 298) to prevent HMR loops. Verify this glob covers `.txt` files too (it should — it unwatches the whole directory, not just `*.json`). No change likely needed, but confirm.

### Todo 7: Verify end-to-end

Run the dev server, open a canvas with a terminal widget, and confirm:
