# WebGL Context Virtualization — Implementation Plan

## Problem
Each terminal widget creates its own WebGL context via ghostty-web. Browsers cap at ~8-16 contexts.
Large canvases with many terminals hit this ceiling causing blank renderers and crashes.

## Approach
Viewport-based context virtualization: only visible/active terminals hold live WebGL contexts.
Offscreen terminals release their renderer and show a frozen snapshot fallback.

## Architecture

### WebGLContextPool (new: `packages/react/src/canvas/WebGLContextPool.jsx`)
- React context + provider wrapping the canvas in CanvasPage
- Lease-based slot management with generation tokens to prevent stale opens
- MAX_LIVE = 6 (conservative, configurable)
- Priority: PINNED (expanded/focused) > VISIBLE > NEAR_VIEWPORT > OFFSCREEN
- Pinned terminals (expanded/selected) are never evicted
- 3-second hysteresis before evicting recently-visible terminals
- API: `useWebGLSlot(widgetId)` → `{ isLive, generation }`
- Pool receives viewport rect + selected/expanded IDs from CanvasPage

### Viewport Visibility (computed in CanvasPage)
- Manual viewport computation in canvas coordinates (avoids IO issues with CSS transform zoom)
- `viewportRect = { x: scrollLeft/scale, y: scrollTop/scale, w: clientWidth/scale, h: clientHeight/scale }`
- Terminal widgets classified as visible/near/offscreen based on position overlap
- Updated on scroll/zoom via requestAnimationFrame throttling

### FrozenTerminalOverlay (new: `packages/react/src/canvas/widgets/FrozenTerminalOverlay.jsx`)
- Renders when terminal is in frozen state (no live WebGL slot)
- Fetches snapshot from existing `/terminal-snapshot/:id` endpoint
- Displays ANSI-rendered HTML (reuses ansi-to-html like TerminalReadWidget)
- "Click to activate" prompt on interaction

### TerminalWidget Changes
- Calls `useWebGLSlot(id)` to get live/frozen status
- When `isLive`: creates ghostty + WS (existing behavior)
- When `!isLive`: disposes ghostty + closes WS, renders FrozenTerminalOverlay
- Uses `generation` as effect dependency for clean re-creation
- Expanded terminals request PINNED priority (non-evictable)

### PromptWidget Changes
- Only requests a slot when output terminal is actually shown (`showOutput`)
- Lightweight: release slot when output is hidden

### CanvasPage Integration
- Wraps canvas children with `<WebGLContextPoolProvider>`
- Computes viewport rect on scroll/zoom events
- Feeds selected widget IDs and expanded state into pool provider
