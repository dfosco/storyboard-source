# Prompt Widget History

## Problem

Prompt widgets have no memory. When a user submits a prompt and resets, the previous text is gone. Users can't recall past prompts and risk losing in-progress text if they accidentally navigate away.

## Approach

Add a per-widget prompt history system that persists to `.storyboard/prompts/{widgetId}.json` and is navigable with ↑/↓ arrow keys (shell-style), with auto-save of the current draft.

## Architecture

### Storage

- **Location:** `.storyboard/prompts/{widgetId}.json`
- **Format:** JSON array of prompt entries, newest last
- **No cleanup / GC** — user will add a garbage collection script later

```json
[
  { "text": "Create a login page", "ts": 1713900000000 },
  { "text": "Add error handling to the form", "ts": 1713900300000 }
]
```

### Server-side

New endpoint on the canvas server:

- **`GET /prompt/history?widgetId={id}`** — returns the full history array
- **`POST /prompt/history`** — appends or updates an entry
  - Body: `{ widgetId, text }` — appends a new entry (deduplicates consecutive identical texts)
- **`POST /prompt/history/draft`** — saves the current draft (auto-save)
  - Body: `{ widgetId, text }` — overwrites the draft slot (separate from submitted history)

The file on disk stores both submitted entries and the current draft:

```json
{
  "entries": [
    { "text": "Create a login page", "ts": 1713900000000 },
    { "text": "Add error handling to the form", "ts": 1713900300000 }
  ],
  "draft": { "text": "Work in progress text...", "ts": 1713901000000 }
}
```

### Client-side (PromptWidget.jsx)

#### History navigation (↑/↓ arrow keys)

- Maintain a `historyIndex` ref (not state — avoids re-renders on every keystroke)
  - `-1` = current draft (default)
  - `0..N` = index into submitted history (0 = oldest, N = newest)
- On **ArrowUp** at the first line of the textarea:
  - If `historyIndex === -1`, save current text as draft, move to newest history entry
  - If `historyIndex > 0`, move to previous (older) entry
  - Replace textarea value with the history entry text
- On **ArrowDown** at the last line of the textarea:
  - If navigating history (`historyIndex >= 0`), move to next (newer) entry
  - If at the newest entry, move to `historyIndex = -1` and restore the draft
- **"First line" / "last line" detection:** Check if cursor is on line 1 (for up) or the last line (for down) by comparing `selectionStart` against newline positions in the text

#### Auto-save draft

- Every 500ms, if `draftText` has changed since last save, POST to `/prompt/history/draft`
- Use a `useEffect` with a 500ms interval + a `lastSavedRef` to debounce
- On component mount, fetch history and restore draft if present

#### On submit

- POST the submitted text to `/prompt/history` (appends to entries)
- Reset `historyIndex` to `-1`

## Notes

- History is per-widget-ID, not per-canvas. If a widget is duplicated, it gets a fresh history (new ID).
- No maximum history length enforced — the user will add GC later.
- The draft is separate from history entries — pressing up from a fresh draft always saves the draft first, and pressing down past the newest entry always restores it.
- The auto-save interval (500ms) means at most 500ms of text can be lost on a crash — acceptable tradeoff vs saving on every keystroke.
