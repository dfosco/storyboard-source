# Plan: CI-based snapshot generation with stable `@latest` naming

## Problem

Snapshot generation runs in the pre-push hook, blocking every `git push` for 30-60+ seconds while Playwright captures screenshots. It also regenerates all missing snapshots indiscriminately — even unchanged widgets get re-evaluated.

## Goal

Move snapshot generation to a **non-blocking GitHub Actions workflow** that runs after the build/deploy completes. Use a **stable naming convention** (`snapshot-{widgetId}--{theme}--latest.png`) so widget props point to a predictable URL that gets overwritten in place.

## Approach

### 1. Stable `@latest` snapshot naming

Change `writeImage()` to produce deterministic filenames:
```
snapshot-{widgetId}--{theme}--latest.png
```

Instead of timestamped filenames. Widget props always reference this URL. The file gets overwritten on each regeneration. No more accumulating old snapshot files.

Update `snapshotLight`/`snapshotDark` props to use the new naming.

### 2. Dirty-widget detection

Add a `--changed-only` flag to `storyboard snapshots` that:
- Runs `git diff HEAD~1 --name-only` to find changed `.canvas.jsonl` files
- For each changed canvas, diffs the materialized widget state to find widgets whose embed-relevant props changed (src, storyId, exportName, url, width, height, zoom)
- Only regenerates snapshots for those widgets
- Skips canvases with no JSONL changes entirely

This makes both local and CI runs fast — only changed widgets get captured.

### 3. GitHub Actions workflow (`snapshots.yml`)

A new workflow that:
- Triggers on `workflow_run` completion of `preview.yml` (non-blocking)
- Checks out the source branch
- Installs deps + Playwright Chromium
- Runs `storyboard snapshots --changed-only`
- If any snapshots were generated, commits them back to the source branch with `[skip ci]`

### 4. Remove snapshot stage from pre-push

The pre-push hook currently runs `stage_snapshots`. Remove it so pushes are fast. CI handles snapshots instead.

## Files to change

| File | Action | Description |
|------|--------|-------------|
| `packages/core/src/cli/snapshots.js` | Modify | Stable naming, `--changed-only` flag, dirty detection |
| `.github/workflows/snapshots.yml` | Create | CI workflow triggered after preview deploy |
| `.githooks/pre-push` | Modify | Remove `stage_snapshots` from STAGES array |

## Steps

### Step 1: Stable snapshot naming in `writeImage`
- Replace timestamped filename with `snapshot-{widgetId}--{theme}--latest.png`
- Update skip logic to check if the file exists on disk (not just props) when deciding freshness

### Step 2: Add `--changed-only` flag
- Parse `--changed-only` from args
- When set, run `git diff HEAD~1 --name-only -- '*.canvas.jsonl'` to find changed canvases
- For each changed canvas, diff the old vs new materialized state to find widgets with changed embed-relevant props
- Only include those widgets in the capture list
- If no canvases changed, exit early with "No changes detected"

### Step 3: Create `snapshots.yml` workflow
- Trigger: `workflow_run` on `preview.yml` completion (success only)
- Extract the branch name from the triggering workflow
- Checkout, install deps, install Playwright Chromium
- Run `npx storyboard snapshots --changed-only`
- If files changed, commit and push with `[skip ci]`
- Use `contents: write` permission

### Step 4: Remove snapshots from pre-push
- In `.githooks/pre-push`, remove `stage_snapshots` from the `STAGES` array
- Keep the hook infrastructure for future stages

## Edge cases & risks

- **First deploy of a new widget**: No snapshot exists yet. The widget falls back to iframe rendering (existing behavior). CI generates the snapshot on next push.
- **Commit-back loop**: The `[skip ci]` in the commit message prevents the snapshot commit from triggering another workflow run.
- **Branch deploys**: The workflow runs per-branch, so each branch gets its own snapshots.
- **Figma embeds**: External URLs can't be captured in CI without auth — skip them in `--changed-only` mode or keep them in local-only mode.
- **Concurrent pushes**: `concurrency` group on the workflow prevents parallel snapshot runs on the same branch.
