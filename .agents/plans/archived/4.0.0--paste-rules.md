# Config-Controlled Paste Rules

## Problem

Paste handling in `CanvasPage.jsx` uses hardcoded if/else chains to decide widget type (figma-embed, prototype, link-preview, markdown). This is brittle and not extensible. We need a config-driven rule engine that:

1. Routes paste content through ordered rules to determine widget type + props
2. Supports `storyboard.config.json` overrides for custom URL patterns
3. Preserves all current default behavior when no config is set
4. Handles branch-prefixed base paths for same-origin detection

## Approach

Extract paste routing into a standalone, testable module (`pasteRules.js`) in `packages/react/src/canvas/widgets/`. The module:

- Exports a `resolvePaste(text, context)` function that returns `{ type, props }` or `null`
- Uses an ordered rule array: each rule has a `match` function and a `resolve` function
- Built-in rules replicate current behavior: image (handled separately), widget-ref, figma, same-origin, fallback URL, markdown
- Config rules from `storyboard.config.json` can be prepended (higher priority) or appended
- `CanvasPage.jsx` calls `resolvePaste()` instead of inline if/else

## Files to change

| Action | File | Description |
|--------|------|-------------|
| Create | `packages/react/src/canvas/widgets/pasteRules.js` | Rule engine + built-in rules |
| Modify | `packages/react/src/canvas/CanvasPage.jsx` | Replace inline paste logic with `resolvePaste()` call |
| Modify | `storyboard.config.json` | Add `canvas.pasteRules` schema docs (no actual rules — empty by default) |

## Steps

1. **Create `pasteRules.js`** with:
   - `createPasteContext(origin, basePath)` — builds context object with helpers (`isSameOrigin`, `extractSrc`, `looksLikeUrl`)
   - Built-in rules array: `widgetRef`, `figma`, `sameOriginPrototype`, `linkPreview`, `markdown`
   - `resolvePaste(text, context, configRules?)` — iterates rules in order, returns first match
   - Config rule format: `{ pattern: string (regex), type: string, propsTemplate?: object }`

2. **Refactor `CanvasPage.jsx`**:
   - Import `createPasteContext` and `resolvePaste` from `pasteRules.js`
   - Move `isSameOriginPrototype`, `extractPrototypeSrc`, `looksLikeWebUrl` into `pasteRules.js` (exported for tests)
   - `handlePaste` calls `resolvePaste(text, ctx)` and uses the returned `{ type, props }`
   - Image paste and widget-ref paste remain as early exits (they need clipboard/canvas API access)

3. **Write tests** for `pasteRules.js` covering:
   - Each built-in rule in isolation
   - Rule precedence (figma > same-origin > link-preview > markdown)
   - Config rules with higher priority than built-ins
   - Branch base-path same-origin detection
   - Null/empty/invalid clipboard text
   - Edge cases: malformed URLs, URLs with special chars

## Edge cases & risks

- Widget-ref paste uses canvas API — it stays in CanvasPage.jsx, not in the rule engine
- Image paste uses `clipboardData.items` — also stays in CanvasPage.jsx
- Config rules with regex patterns need careful validation to avoid ReDoS
- `BRANCH_PREFIX_RE` must match both in the rule engine and existing code
- Must preserve exact same widget props (width, height, etc.) as current code
