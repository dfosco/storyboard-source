# Widget Perf Snap 2 — Embed UX Fixes

## Goals
1. Empty-state embeds show "Click to open" instead of "Click to interact"
2. Eliminate flickering during embed loading transitions (placeholder → snapshot → iframe)
3. Replace dual-img snapshot approach with single theme-aware `<img>`, eliminating visibility-swap flash

## Approach

### Fix 1: Empty state label
- Pass `hasAnySnap` to the overlay in both PrototypeEmbed and StoryWidget
- When `!hasAnySnap`, overlay text = "Click to open"; otherwise "Click to interact"

### Fix 2: Flickering
- Add `transition: opacity 200ms ease` to snapshot images in CSS
- Ensure iframe starts `opacity: 0` and transitions to `opacity: 1` on load
- Keep snapshot visible underneath iframe (already stacked via absolute positioning)

### Fix 3: Single snapshot node
- Replace two `<img>` tags (light + dark with visibility toggling) with one `<img>`
- Compute `src` from `isDark`: dark theme → snapshotDark (fallback snapshotLight), light → snapshotLight (fallback snapshotDark)
- `resolveCanvasTheme()` already handles prefers-color-scheme + localStorage — just ensure matchMedia listener triggers re-resolve
- Add `matchMedia('(prefers-color-scheme: dark)')` change listener to PrototypeEmbed and StoryWidget to trigger theme re-read
