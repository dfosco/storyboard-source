# 4.1.0 — Storyboard Dev Server & Branch Switching

## Problem

Storyboard's dev server (`npx storyboard dev`) runs Vite as a child process, and the `/_storyboard/` API middleware runs **inside** that Vite process. This means:

1. **Branch switching can't work** — killing the current dev server to start a new one would kill the API that initiated the switch
2. **No persistent orchestration** — there's no parent process that can manage multiple dev servers or restart them
3. **The two-way server (browser ↔ source code)** was always meant to be a standalone node server, not embedded in Vite

## Proposed Architecture

```
┌────────────────────────────────────────────────────┐
│  Storyboard Server (Node.js)                       │
│  - Persistent process, not tied to any branch      │
│  - Manages child Vite dev processes                 │
│  - Serves /_storyboard/ API (branch switching,     │
│    workshop, canvas, comments, worktrees)           │
│  - Proxies requests to the active Vite server       │
│  - Manages Caddy routes                            │
└───────────┬────────────────────────────────────────┘
            │ spawns/manages
    ┌───────┴───────┐
    │ Vite Dev      │  ← one per active branch
    │ (child proc)  │
    │ port: 1235+   │
    └───────────────┘
```

### Key changes

1. **`storyboard server`** (new command) — long-running Node.js process that:
   - Starts Vite dev servers as child processes
   - Serves `/_storyboard/` API on its own port (or proxied through Caddy)
   - Handles `POST /_storyboard/switch-branch` — stops current Vite, starts new one, responds with redirect URL
   - Can manage multiple simultaneous Vite servers if needed

2. **`storyboard dev`** (updated) — becomes a thin wrapper that:
   - Starts the server if not running
   - Tells the server to start a dev session for the requested branch
   - Opens the browser

3. **`/_storyboard/` middleware** (moved) — extracted from `server-plugin.js` into the standalone server. Workshop, canvas create, autosync, comments, worktrees endpoints all live here.

4. **Branch switching flow**:
   ```
   Browser: POST /_storyboard/switch-branch { branch: "feature-x" }
   Server:  1. Check if branch worktree exists, create if not
            2. Check if Vite is running for that branch (port health check)
            3. If not, spawn `vite --port {port}` for that branch
            4. Wait for Vite ready (poll port)
            5. Update Caddy route
            6. Respond with { url: "http://storyboard-core.localhost/branch--feature-x/" }
   Browser: window.location.href = response.url
   ```

## Phases

### Phase 1: Standalone Server
- Create `packages/core/src/server/index.js` — the persistent Node server
- Move `/_storyboard/` route handlers from `server-plugin.js` to the new server
- Keep `server-plugin.js` as a thin proxy that forwards `/_storyboard/` requests to the standalone server
- `storyboard server` CLI command

### Phase 2: Branch Switching API
- `POST /_storyboard/switch-branch` endpoint
- Vite child process management (spawn, health check, kill)
- Caddy route updates
- Wire BranchBar.svelte and ViewfinderNew.jsx to call the API

### Phase 3: Multi-branch Support
- Run multiple Vite servers simultaneously (one per active branch)
- Process lifecycle management (auto-stop idle servers)
- Status dashboard in Viewfinder

## Notes

- The BranchBar Svelte component and worktrees API endpoint already exist from 4.0.0--viewfinder-new
- Caddy proxy infrastructure already handles multi-branch routing
- ports.json already tracks branch→port mappings
- The server should be able to run without Caddy (direct port access fallback)
