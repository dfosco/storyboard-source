# Selected Widgets ‚Äî Real-time Canvas Context for Copilot

## Goals

1. **Real-time widget context** ‚Äî Create `.selectedwidgets.json` at repo root that reflects the currently focused canvas and selected widgets in real-time, enabling Copilot to understand what the user is looking at/working with.
2. **Multi-tab awareness** ‚Äî Handle multiple browser tabs gracefully: switching tabs updates the active canvas; restoring focus to a tab restores its selection state.
3. **Agent integration** ‚Äî Update `AGENTS.md` with foundational instructions so Copilot reads `.selectedwidgets.json` on every prompt and uses it as implicit context (e.g., "implement this image" ‚Üí the selected image widget).

## Problem Statement

When users interact with Storyboard canvases, Copilot has no awareness of what's visually selected in the browser. Users must manually describe which widgets they're referring to. This feature creates a live bridge: browser selection ‚Üí file on disk ‚Üí Copilot context.

## Approach

**Client-side (CanvasPage.jsx):**
- On canvas mount + browser tab focus: send `storyboard:canvas-focused` HMR event with `canvasId` and current selection
- On `selectedWidgetIds` change: debounce 500ms, then send `storyboard:selection-changed` HMR event with `canvasId`, widget IDs, and full widget data (type, props)
- On canvas unmount: send unfocus event
- On `visibilitychange`/`focus`: re-send current state so tab switches restore selection

**Server-side (new `selectedWidgets.js` module, wired in `server-plugin.js`):**
- Listen for HMR events, track active client (by Vite WebSocket client identity)
- On focus: set active client, write `.selectedwidgets.json` with canvas info + current selection
- On selection change: if from active client, write updated file
- On disconnect/cleanup: if active client disconnects, clear the file
- Unwatch `.selectedwidgets.json` to prevent HMR loops

**File format:**
```json
{
  "canvasId": "design-system",
  "canvasFile": "src/canvas/design-system.canvas.jsonl",
  "selectedWidgetIds": ["img-abc123", "sticky-def456"],
  "widgets": [
    { "id": "img-abc123", "type": "image", "props": { "src": "screenshot.png" } },
    { "id": "sticky-def456", "type": "sticky-note", "props": { "text": "Todo" } }
  ]
}
```

## Steps

1. **Create `selectedWidgets.js`** ‚Äî Server module that:
   - Exports `setupSelectedWidgets(server, root)` 
   - Listens for `storyboard:canvas-focused` and `storyboard:selection-changed` via `server.hot.on()`
   - Tracks active client (last to send focus)
   - Writes `.selectedwidgets.json` atomically (write to `.selectedwidgets.json.tmp` then rename)
   - Resolves `canvasId` ‚Üí file path using `findCanvasPath()`
   - On server close, cleans up the file
   - Periodic cleanup: if active client disconnects, clear file

2. **Wire in `server-plugin.js`** ‚Äî In `configureServer`:
   - Import and call `setupSelectedWidgets(server, root)`
   - Add `server.watcher.unwatch('.selectedwidgets.json')` to prevent HMR loops

3. **Client-side in `CanvasPage.jsx`** ‚Äî New `useEffect`:
   - On mount: send `storyboard:canvas-focused` with `{ canvasId, widgetIds: [], widgets: [] }`
   - On `selectedWidgetIds` change: debounce 500ms, send `storyboard:selection-changed` with `{ canvasId, widgetIds: [...ids], widgets: matchingWidgetData }`
   - On `visibilitychange` (document becomes visible) or `window.focus`: re-send focus + current selection
   - On unmount: send `storyboard:canvas-unfocused`

4. **Update `.gitignore`** ‚Äî Add `.selectedwidgets.json`

5. **Update `AGENTS.md`** ‚Äî Add section at the very top of "General instructions":
   - Before processing any prompt, read `.selectedwidgets.json`
   - If widgets are selected, treat them as implicit context
   - For image widgets: resolve `props.src` to the actual image file and load it into context
   - Include the canvas file path for cross-referencing

## Edge Cases & Risks

- **HMR loop**: Writing `.selectedwidgets.json` could trigger Vite watcher ‚Üí solved by unwatching the file
- **Stale client**: A tab crashes without sending unfocus ‚Üí solved by periodic cleanup checking `server.ws.clients`
- **Race conditions**: Rapid tab switching ‚Üí last-writer-wins (the latest focus event always wins)
- **Non-canvas pages**: If the user navigates away from a canvas, the unfocus event clears the file
- **Dev-only**: `import.meta.hot` guard ensures no events are sent in production builds
- **File atomicity**: Write to temp file + rename avoids partial reads by Copilot
