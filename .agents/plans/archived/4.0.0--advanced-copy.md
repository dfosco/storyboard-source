# Advanced Copy — `canvasId/widgetId` clipboard format

## Problem
- **Shift+C** copies only the bare widget ID — doesn't include canvas context, and doesn't work for component widgets (they aren't in `localWidgets`)
- **Cmd+C** copies URL/content — useful but not for duplicating widgets
- **Cmd+V** creates new widgets from clipboard content — no way to duplicate an existing widget by pasting

## Approach

Change the copy/paste shortcuts in `CanvasPage.jsx`:

1. **Shift+C** → Copy the widget ID (any widget, including components). Works for any selected widget. Format: bare `widgetId`.
2. **Cmd+C** → Copy `canvasName/widgetId` to clipboard. This is the "duplicate reference" format.
3. **Cmd+V** → If clipboard contains `canvasName/widgetId`, look up the widget (same canvas from local state, or cross-canvas via `canvasApi` GET), then duplicate it centered on viewport. Falls through to existing paste logic if format doesn't match.

## Files to Change

- `packages/react/src/canvas/CanvasPage.jsx` — keyboard handler + paste handler
- `packages/react/src/canvas/canvasApi.js` — add `getCanvas(name)` for cross-canvas fetch

## Steps

1. Add `getCanvas(name)` export to `canvasApi.js`
2. In `CanvasPage.jsx` keyboard handler:
   - **Shift+C**: Remove `size === 1` restriction for components — but actually it already requires size===1 which is fine. The real bug is it searches `localWidgets` which doesn't have component widgets. Fix: if widget not found in `localWidgets`, check if ID starts with `jsx-` (it's a component). Always copy `widgetId` (not file path for images — that was the old behavior).
   - **Cmd+C**: Change to copy `${name}/${widgetId}` instead of URL/content.
3. In paste handler: before existing logic, check if text matches `canvasName/widgetId` pattern. If so:
   - If `canvasName === name`, find widget locally (in `localWidgets` or component sources)
   - Otherwise, fetch canvas via API and find widget
   - Duplicate widget at viewport center via `addWidgetApi`
   - Component widgets (`jsx-*`) cannot be duplicated cross-canvas (they're code, not data) — show nothing / skip silently

## Edge Cases
- Component widgets can be shift+C copied but cannot be cmd+V duplicated (they're JSX, not JSON widgets)
- Cross-canvas paste: need network fetch — handle errors gracefully
- Clipboard text that happens to contain `/` but isn't a widget ref — use strict validation
