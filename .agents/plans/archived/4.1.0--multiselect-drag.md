# Marquee Multi-Select Drag

## Goals

1. When the user clicks and drags on the canvas background (not on a widget, not space+drag panning), a translucent selection rectangle appears and follows the cursor.
2. On mouse-up, any widgets whose bounding boxes intersect with the selection rectangle become selected.

## Problem

Canvas currently supports multi-select only via shift+click. There's no way to select multiple widgets by dragging a rectangle over them — a standard pattern in design tools and OS desktops.

## Approach

Create a `useMarqueeSelect` hook that handles all the marquee logic, keeping CanvasPage changes minimal. The hook attaches mousedown/mousemove/mouseup listeners to the scroll container, draws a selection rectangle overlay, and computes widget intersection on mouse-up.

## Files to Change

- **`packages/react/src/canvas/useMarqueeSelect.js`** — NEW hook with all marquee logic
- **`packages/react/src/canvas/MarqueeOverlay.jsx`** — NEW component rendering the selection rectangle
- **`packages/react/src/canvas/CanvasPage.module.css`** — Add marquee rectangle styles
- **`packages/react/src/canvas/CanvasPage.jsx`** — Wire up the hook, pass widget data for intersection

## Steps

1. Create `useMarqueeSelect.js` hook:
   - On mousedown on canvas background (not widget, not space held): record start point in canvas coords
   - On mousemove: update end point, compute rectangle in screen coords for overlay
   - On mouseup: compute final rectangle in canvas coords, test intersection with all widgets, call `setSelectedWidgetIds`
   - Return `{ marqueeRect, onMouseDown }` — rect is null when inactive

2. Create `MarqueeOverlay.jsx`:
   - Renders a fixed-position div when `marqueeRect` is non-null
   - Styled with translucent blue fill, thin blue border

3. Add CSS for marquee in `CanvasPage.module.css`

4. Wire into `CanvasPage.jsx`:
   - Call `useMarqueeSelect`, pass widget positions, zoom, scrollRef, setSelectedWidgetIds
   - Replace `handleDeselectAll` onClick with marquee's mousedown handler
   - Render `MarqueeOverlay` inside scroll container

## Edge Cases & Risks

- Must not conflict with space+drag panning — check `spaceHeld` before starting marquee
- Must not trigger on widget clicks — mousedown target must be the canvas background
- Must account for zoom scale when converting screen↔canvas coordinates
- Must handle scroll during drag (scroll container offset changes)
- Shift+marquee should add to existing selection (stretch goal, skip for v1)
