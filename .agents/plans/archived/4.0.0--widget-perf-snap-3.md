# Figma Embed Fixes — widget-perf-snap-3

## Problem
Figma embeds have three UX issues:
1. Iframe unloads immediately when user clicks outside — should retain loaded state for 5 minutes
2. Iframe only starts loading on user click — should start loading on page load since Figma is slow
3. Empty state (no URL) is plain text — should show a feather-icons figma icon

## Approach
All changes are in `FigmaEmbed.jsx` and `FigmaEmbed.module.css`.

## Files to Change
- `packages/react/src/canvas/widgets/FigmaEmbed.jsx` — all 3 fixes
- `packages/react/src/canvas/widgets/FigmaEmbed.module.css` — empty state styling

## Steps

### 1. Load iframe on mount
- Change `useState(false)` → `useState(true)` for `showIframe`
- Iframe loads immediately behind the interaction overlay

### 2. Retain open state for 5 minutes
- Add `teardownTimerRef` and `exitSessionRef` refs
- On deselect (pointerdown outside): start 5-min timer instead of immediate `setShowIframe(false)`
- On re-enter interactive (`enterInteractive`): cancel timer, increment session counter
- On unmount: clear teardown timer
- Pattern matches PrototypeEmbed's keep-alive implementation

### 3. Empty state with figma icon
- Add inline `FigmaIcon` component using feather-icons figma SVG (monochrome stroke-based)
- Replace `<p>No Figma URL</p>` with icon + "No Figma URL" text
- Add `.emptyState` and `.emptyIcon` CSS classes

## Edge Cases
- Timer cleanup on unmount prevents stale setState calls
- exitSessionRef counter prevents stale timer callbacks from firing after re-interaction
- Empty state is purely visual — no functional risk
