# Canvas Folder Iteration

## Goal
Refine the multi-page canvas system: canvas directories get `.meta.json` metadata, the title bar is simplified (no editing), and "Local Editing" label flows inline with the PageSelector.

## Changes

### 1. Canvas folder `.meta.json` support
- `src/canvas/examples/examples.meta.json` — metadata file with title, description, author
- `data-plugin.js` — discovers `.meta.json` files, injects `_canvasMeta` into canvas entries
- `viewfinder.js` — uses `_canvasMeta` for canvas group display
- `context.jsx` — passes meta through `canvasGroupMap` to PageSelector

### 2. Server canvas folder support  
- `POST /create` — supports creating pages in plain directories (not just `.folder/`), auto-creates `.meta.json`
- `GET /list` — includes `groups` field with metadata from `.meta.json` files
- `GET /folders` — lists plain canvas directories alongside `.folder/` directories

### 3. Title bar simplification
- Removed title editing (always read-only `<h1>`)
- Moved PageSelector inside title bar flex row
- "Local editing" label flows after PageSelector or title
- Removed unused CSS (canvasTitleWrap, canvasTitleMeasure, canvasTitleInput)

### 4. Single-page canvas
- No dropdown picker shown (PageSelector returns null for <2 pages)
- Just shows static title + "Local editing" label
