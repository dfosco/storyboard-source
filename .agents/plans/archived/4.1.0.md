# 4.1.0 — Canvas Page Management UI

## Summary

Restore and enhance the canvas page management experience: editable canvas/folder names, page rename, drag-and-drop reorder, visual separators, and a more spacious dropdown.

## Completed

### Server APIs (`packages/core/src/canvas/server.js`)
- **`PUT /rename-page`** — Renames a `.canvas.jsonl` file on disk, appends `settings_updated` event, updates `.page-order.json` transactionally. Returns new canonical ID and route.
- **`PUT /reorder-pages`** — Writes `.page-order.json` to the canvas folder directory. Accepts flat array of canonical page IDs and `sep-*` separator IDs.
- **`GET /page-order?folder=...`** — Reads `.page-order.json` from a folder, returns `{ order: [...] }` or `{ order: null }`.
- **`PUT /update-folder-meta`** — Updates the `title` field in a folder's `.meta.json`.

### Client API (`packages/react/src/canvas/canvasApi.js`)
- `renamePage(canvasId, newTitle)`
- `reorderPages(folder, order)`
- `getPageOrder(folder)`
- `updateFolderMeta(folder, title)`

### Editable Canvas/Folder Name (`CanvasPage.jsx`)
- `CanvasTitleEditable` component always renders (removed `siblingPages.length > 1` guard).
- Double-click to edit in dev mode. Folder canvases update `.meta.json` via `updateFolderMeta`; single canvases use `updateCanvas` settings.
- Enter commits, Escape cancels, blur commits.

### Page Rename (`PageSelector.jsx`)
- Double-click a page row to enter inline edit mode.
- 300ms click timer (mouse only) distinguishes single-click navigation from double-click edit. Keyboard Enter/Space navigates immediately.
- Validates no duplicate page names (case-insensitive).
- Uses `sb-pending-navigate` + `vite:beforeFullReload` HMR pattern for post-rename navigation.

### Drag & Reorder (`PageSelector.jsx`)
- 6-dot drag grip icon on the left of each page row, visible on hover.
- Native HTML5 drag-and-drop with drop indicator line.
- Drag suppresses click timer to prevent accidental navigation.
- Order persisted via `reorderPages(folder, order)`.

### Separators
- Typing `---` in "Add new page" inserts a visual separator (no file created).
- Separators stored as `sep-{timestamp}` entries in `.page-order.json`.
- Separators are draggable and reorderable alongside pages.
- Badge count (`x/y`) only counts real pages.

### Dropdown Styling (`PageSelector.module.css`)
- `min-width: 260px` (was 180px), `max-width: 360px` (was 300px)
- `padding: 6px` (was 4px)
- Items use flex layout with gap for drag handle
- Added `.dragHandle`, `.itemContent`, `.itemDragging`, `.dropIndicator` classes

## Bug Fixes
- Fixed `.page-order.json` rename handler — was checking `entry.type === 'page'` but the file stores flat strings (canonical IDs). Now matches and replaces correctly.
