# Component Refoldering Plan

## Problem
Flat component files in `src/components/` violate the convention that every component must live in its own directory (`ComponentName/ComponentName.ext`). Some components also lack stories.

## Components to Move

| Component | Files | Has Story? | Importers |
|-----------|-------|-----------|-----------|
| ColorModeSwitcher | .jsx, .module.css | ❌ | Example/index.jsx |
| FormContext | .jsx | ❌ (internal) | StoryboardForm, TextInput, Textarea |
| GlobalNavigation | .jsx, .module.css | ❌ | Application.jsx (×2) |
| Playground | .jsx, playground.module.css→Playground.module.css | ❌ | None |
| SidebarNavigation | .jsx | ❌ | Application.jsx (×2) |
| StoryboardForm | .jsx | ❌ | Example/index.jsx |
| TextInput | .jsx + text-input.story.jsx | ✅ (but imports Primer, not local component) | Example/index.jsx |
| Textarea | .jsx + textarea.story.jsx | ✅ (but imports Primer, not local component) | Example/index.jsx |
| ThemeSync | .jsx | ❌ | None |

**AppSidebar** — already in a folder ✅

## Approach

### Phase 1: Move files into folders
For each component, create `src/components/ComponentName/` and move files in.

### Phase 2: Fix internal imports
- FormContext imports from StoryboardForm, TextInput, Textarea: `./FormContext.jsx` → `../FormContext/FormContext.jsx`
- CSS imports: update relative paths
- Playground: rename `playground.module.css` → `Playground.module.css`

### Phase 3: Fix external imports
- `@/components/ColorModeSwitcher.jsx` → `@/components/ColorModeSwitcher/ColorModeSwitcher.jsx`
- Same pattern for GlobalNavigation, SidebarNavigation, StoryboardForm, TextInput, Textarea

### Phase 4: Fix/create stories
- TextInput story: update to import local TextInput component
- Textarea story: update to import local Textarea component  
- Generate stories for: ColorModeSwitcher, GlobalNavigation, Playground, SidebarNavigation, StoryboardForm, ThemeSync
- FormContext is internal-only, skip story

### Phase 5: Update skills
- The create skill already documents the folder pattern correctly
- Verify no other skills reference flat component paths
