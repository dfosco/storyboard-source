# Terminal Buffer Rearchitecture Plan

## Problem

Today the terminal buffer system has limited capture:
- In-memory `rawTailBuffers` map capped at 2,000 chars
- `captureSnapshot()` polls `tmux capture-pane` every 5s, writes to `.storyboard/terminal-snapshots/<canvasId>/<widgetId>.json`
- Snapshots are in `.storyboard/` which is gitignored — invisible to production and other agent sessions
- No API for agents to read each other's terminal output
- No scrollback beyond the visible pane + 2KB raw tail

## Requirements

### 1. Dual-file buffer architecture

| File | Path | Visibility | Scrollback cap |
|------|------|-----------|----------------|
| **Buffer** (private) | `.storyboard/terminal-buffers/<widgetId>.buffer.json` | gitignored, agent-readable | 5 minutes |
| **Snapshot** (public) | `assets/.storyboard-public/terminal-snapshots/<widgetId>.snapshot.json` | committed, production-visible | 1 minute |

### 2. API endpoint / CLI command

- `GET /_storyboard/canvas/terminal-buffer/<widgetId>?length=N`
- `storyboard terminal read <widgetId> [--length N]`

### 3. Terminal `private` action

- When private → skip public snapshot + rename existing to `~<filename>` (gitignored)
- When un-privated → rename back

### 4. Scrollback capture improvements

- Time-windowed rolling buffer replaces 2K char cap

## Decisions

- `terminalID` = `widgetId`
- ANSI: kept in buffer, stripped from public snapshot
- Auth: localhost-only
- Migration: old paths kept as fallback reads
