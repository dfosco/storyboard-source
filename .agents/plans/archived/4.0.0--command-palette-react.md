# Port CommandPalette to @dfosco/storyboard-react

## Goals

1. Move the CommandPalette React component from the consumer app (`src/components/CommandPalette/`) into `packages/react/src/CommandPalette/` so all storyboard consumers get it automatically.
2. Export it from `@dfosco/storyboard-react` and update the consumer app to import from the package instead.
3. Add `react-cmdk` as a dependency of `@dfosco/storyboard-react`.

## Approach

Copy the three files (CommandPalette.jsx, CreateDialog.jsx, command-palette.css) into the react package, update imports to use relative `@dfosco/storyboard-core` paths, export from index.js, add `react-cmdk` dep, and update the consumer app to import from the package.

## Files to change

- **Create** `packages/react/src/CommandPalette/CommandPalette.jsx`
- **Create** `packages/react/src/CommandPalette/CreateDialog.jsx`
- **Create** `packages/react/src/CommandPalette/command-palette.css`
- **Edit** `packages/react/src/index.js` — add export
- **Edit** `packages/react/package.json` — add `react-cmdk` dependency
- **Edit** `src/index.jsx` — import from `@dfosco/storyboard-react` instead of local
- **Delete** or deprecate `src/components/CommandPalette/` (consumer copy)

## Steps

1. Add `react-cmdk` to packages/react/package.json dependencies
2. Copy the 3 component files into packages/react/src/CommandPalette/
3. Export `StoryboardCommandPalette` from packages/react/src/index.js
4. Update src/index.jsx to import from the package
5. Remove the consumer-app copy
6. Run lint/build/test

## Edge cases

- `react-cmdk` brings Tailwind CSS — should work since consumer already has Tailwind
- `import.meta.env.BASE_URL` usage in isHiddenInPalette — works in Vite contexts only (fine for our use case)
